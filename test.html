Tester un site pour le SEO

1. Vérifier l’indexation et les balises

Commence par tester si ton site est indexé : site:monsite.com dans Google. Ensuite, vérifie les balises essentielles avec la console du navigateur : 

document.querySelector('title').textContent
document.querySelector('meta[name="description"]').getAttribute('content')
Assure-toi que chaque page a un titre unique et une meta description descriptive.

2. Analyse technique avec Lighthouse

Lighthouse (Chrome DevTools ou extension) permet de tester performance, accessibilité et SEO. Exemple pratique : 1. Ouvre DevTools (F12) → Audits → SEO → Générer le rapport. 2. Corrige les erreurs détectées : images sans alt, balises manquantes, temps de chargement trop long.

Pour tester les temps de réponse serveur : 

curl -I https://monsite.com/page
Vérifie le status, le Content-Type et les directives cache. Ces mesures impactent directement le référencement.

3. Tester la structure et les liens

Vérifie les liens internes et externes. Snippet JS pour lister tous les liens sur la page : 


const liens = document.querySelectorAll('a');
liens.forEach(a => console.log(a.href));
Cela permet de repérer les liens brisés ou redondants. Bonus : installe l’extension Chrome
Check My Links Chrome.
Check My Links Firefox

Exercice pratique

1. Vérifie l’indexation de ton site avec site:monsite.com. 2. Ouvre Lighthouse et corrige toutes les erreurs SEO détectées. 3. Liste tous les liens et identifie ceux à corriger.
